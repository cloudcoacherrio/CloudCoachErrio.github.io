Overview

This step-by-step guide will help you create a data streaming pipeline using AWS services. The approach uses a combination of local Python scripts and the AWS Management Console to set up your resources. You'll download Python scripts from GitHub and execute some locally while setting up Lambda functions through the AWS console.

Prerequisites

Before starting, make sure you have:

  • - An AWS account (free tier is sufficient)
  • - A Mac computer with internet connection
  • - Basic knowledge of how to use Terminal on Mac

Detailed Setup Instructions

1. Setting Up Your Mac Environment

Install Python 3 (if not already installed)

  1. a. Open Terminal (find it in Applications > Utilities or use Spotlight search with Command+Space)
  2. b. Check if Python is installed by typing: python3 --version
  3. c. If not installed, download the installer from python.org and follow the installation steps

Install AWS Command Line Interface (CLI)

  1. a. Open Terminal
  2. b. Install AWS CLI using pip (Python's package manager): run the code below
  3. - pip3 install awscli
  4. c. Verify installation:run the code below 
    - aws --version
    - You should see version information displayed

Set Up AWS Credentials

  1. a. Sign in to your AWS Management Console
  2. b. Click on your username at the top right, then select "Security credentials"
  3. c. Under "Access keys", click "Create access key"
  4. d. Follow the prompts, and make sure to download or save your Access Key ID and Secret Access Key
  5. e. In Terminal, configure AWS CLI: 
    - aws configure
  6. f. Enter your Access Key ID, Secret Access Key, preferred region (like "us-east-1"), and output format (type "json")

Install Required Python Package

  1. a. In Terminal, install boto3 (AWS SDK for Python): 
    - pip3 install boto3

Download Project Files from GitHub

  1. a. Create a folder for the project:
  2. - mkdir aws-kinesis-demo
  3. - cd aws-kinesis-demo
  4. b. Download the Python scripts from the GitHub repository (replace with the actual GitHub URL)
  5. # You might use curl, wget, or simply download from your browser and move the files
    6. # For example: curl -O https://github.com/cloudcoacherrio/kinesis-basic-demo.git <== Repo Files HERE<==

2. Create AWS Resources Locally

Create Kinesis Data Stream

  1. a. In Terminal, navigate to your project folder and run: 
    - python3 createkds.py
  2. b. You should see a message indicating that the stream is being created
  3. c. Wait a few minutes for the stream to become active

Create DynamoDB Table

  1. a. In Terminal, run: 
    - python3 createkdsbasictable.py
  2. b. You should see a message indicating that the table is being created
  3. c. Wait a few minutes for the table to become active

Create IAM Role for Producer Lambda

  1. a. In a new browser tab, navigate to the IAM service
  2. b. Click on "Policies" in the left sidebar
  3. c. Click "Create policy"
  4. d. Click on the JSON tab
  5. e. Copy the content from the producerlambda.json file (from GitHub)
  6. f. Make sure to replace REGION with your AWS region (e.g., "us-east-1")
  7. g. Paste the Json content on the new policy, name the policy, and save.
  8. h. Click on "Roles" in the left sidebar
  9. i. Click the "Create role" button
  10. j. Under "Trusted entity type," select "AWS service"
  11. k. Under "Use case," select "Lambda"
  12. l. Click "Next" at the bottom right
  13. m. In the search box, type the name of the policy you just previously created
  14. n. Check the box next to each policy you need
  15. o. Click "Next" at the bottom right
  16. p. Enter a role name that describes its purpose
  17. q. Review and Click "Create role" at the bottom right

3. Create Lambda Functions in AWS Console

Create Producer Lambda Function

  1. a. Navigate to the Lambda service
  2. b. Click "Create function"
  3. c. Select "Author from scratch"
  4. d. Enter a name (e.g., "KinesisProducerLambda")
  5. e. For Runtime, select "Python 3.9" (or newer version)
  6. f. Under "Permissions", expand "Change default execution role"
  7. g. Select "Use an existing role"
  8. h. In the dropdown, select the name of the role you created for the producer Lambda function
  9. i. Click "Create function"
  10. j. In the function code editor, delete any default code
  11. k. From the GitHub repository, open kdsbasicproducer.py in your browser or local editor
  12. l. Copy the entire code and paste it into the Lambda code editor
  13. m. Click "Deploy" to save your function

  1. a. Navigate to the Lambda service
  2. b. Click "Create function"

Use Case Scenario

Patient Vital Signs Monitoring System Using AWS Streaming Architecture

HealthStream Technologies is developing a real-time patient monitoring solution for hospitals that tracks vital signs from multiple medical devices. The system needs to capture, process, and store data from thousands of medical devices across multiple hospital wards, providing critical alerts to medical staff when readings fall outside safe thresholds.

Business Challenge

Healthcare providers need to:
- Monitor patients' vital signs in real-time (heart rate, blood pressure, temperature, oxygen levels)
- Process thousands of readings per second from medical devices
- Store historical data for analysis and patient records
- Generate immediate alerts for critical conditions
- Ensure data reliability with minimal downtime

Technical Solution

Using AWS Kinesis Data Streams, Lambda, and DynamoDB, we can build a scalable, real-time data processing pipeline that meets these requirements:
1. Data Collection: Medical devices publish vital sign readings to a Kinesis Data Stream
2. Processing: Lambda functions process incoming data to:
- Filter and transform readings
- Detect anomalies in patient vital signs
- Generate alerts for critical conditions
3. Storage: DynamoDB stores processed readings for quick access by healthcare applications

Sample Data Flow

1. A patient's heart rate monitor sends a reading: {"patientId": "P12345", "value": 99, "type": "heartRate", "timestamp": "2023-11-01T14:32:10Z"}
2. The Producer Lambda function receives this data and publishes it to the Kinesis Data Stream
3. The Consumer Lambda function processes this reading:
- Determines if the heart rate is within normal range (60-100 BPM)
- Adds metadata like ward location and attending physician
- Stores the enriched data in DynamoDB
- If the reading is abnormal, flags it for immediate attention
4. Healthcare staff can access current and historical vital signs through their monitoring dashboard
